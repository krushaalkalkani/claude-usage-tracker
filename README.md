# Claude Usage Tracker 📊

A beautifully minimal, fully client-side React desktop dashboard that connects to an undocumented internal Anthropic API to read your **Claude 5-hour** and **7-day** rate limits in real time. 

It automatically polls your actual usage statistics in the background (every 2 minutes) and displays live gauges, a Burn Rate estimator, and historical charts, preventing you from hitting usage caps unexpectedly during intense coding sessions.

## 🚀 The Discovery
This project is built around a community-discovered internal endpoint (`https://api.anthropic.com/api/oauth/usage`) that powers the official `claude-code` CLI tool. By proxying requests to this API, we can extract your account's precise percentage-based utilization without relying on guesswork.

---

## ✨ Features

- **Live Gauges**: Instantly monitor your **5-hour current session** and **7-day weekly limits**.
- **Burn Rate Estimator**: Acts like a speedometer for your usage. Custom algorithm detects if you are chatting heavily and calculates a live `+X% / min` velocity metric so you know exactly how fast you are draining your quota.
- **Historical Data Charts**: Visualizes your session percentage over time using elegant area charts, recording your poll history.
- **Auto-Refreshing & Force Fetch**: Polls Anthropic's OAuth API every 120 seconds automatically with a visual countdown timer. Includes a manual `[Force Fetch]` button for instant refreshes.
- **Minimalist Design**: Built upon a sleek, Vercel/Linear-inspired dark mode aesthetic with ultra-clean `<svg>` line-art and monospace typography.
- **Privacy First**: Completely client-side. Your OAuth token is saved securely in your browser's local storage and is *never* sent to a third-party server (only directly to Anthropic's API via a local Vite proxy).

## 🛠 Tech Stack

- **React** (State management & UI)
- **Vite** (Build tool & CORS API Proxy)
- **Recharts** (Responsive SVG data visualization)
- **Vanilla CSS** (No heavy dependencies, pure minimal system fonts)

## 💻 Getting Started

### Prerequisites
- Node.js installed on your machine.
- An active Anthropic account.

### 1. Clone & Install
```bash
git clone https://github.com/YourUsername/claude-usage-tracker.git
cd claude-usage-tracker
npm install
```

### 2. Start the Local Server
```bash
npm run dev
```
The app will now be running at `http://localhost:5173`. Make sure you let it run here, because the Vite dev server manages the internal proxy to bypass Anthropic's CORS policies securely.

## 🔑 How to Get Your OAuth Token

When you first open the app, you will be prompted for an access token. Because this uses an undocumented API, standard API keys (`sk-ant-api03...`) **will not work**. You need an **OAuth Session Token**.

### The Best Way: Claude Code CLI
If you use Anthropic's official `claude-code` CLI, you already have an OAuth token saved on your machine!

1. If you haven't already, install it: `npm i -g @anthropic-ai/claude-code`
2. Run `claude` once in your terminal and follow the oauth login steps in your browser.
3. Once authenticated, run this command to easily extract the token:

**For Mac Users:**
```bash
security find-generic-password -s "Claude Code-credentials" -w 2>/dev/null | python3 -c "import sys,json; print(json.loads(sys.stdin.read())['claudeAiOauth']['accessToken'])"
```

**For Windows/Linux Users (Finding the file manually):**
Look for the global `claude.json` config file generated by `claude-code` (usually in `~/.claude.json` or `%APPDATA%`), and copy the `accessToken` under the `claudeAiOauth` object.

Paste that token into the Tracker UI!

## ⚠️ Important: Rate Limiting & 429 Errors

Because `/api/oauth/usage` is an undocumented internal API, Anthropic applies **very aggressive rate limiting** to it to prevent spam.

- The app is hard-coded to poll every **120 seconds**. This is typically slow enough to stay under the radar.
- If you click the `Force Fetch` button too many times in a row, Anthropic *will* temporarily ban your token, and the app will show a red `429 Too Many Requests / Rate Limited` error.
- **If you get Rate Limited:** Don't panic! Simply let the app sit for 5 to 15 minutes. The block will automatically expire, and the dashboard will reconnect. Alternatively, you can generate a fresh OAuth token to bypass it immediately.
